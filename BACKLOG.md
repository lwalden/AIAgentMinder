### B-001: Make version-bump.sh Windows-resilient or document it as a Linux/CI-only helper

During the 5.1.3 release, `bash bin/version-bump.sh 5.1.3` hung on Windows: the
`bash` that Node/PowerShell spawns is WSL, and `sed -i` against the repo on the
9p-mounted Windows filesystem stalled (exit 255 after kill). The four version
files were edited directly via the editor instead — identical end state.

Options to consider:
- Replace `sed -i` with a temp-file + `mv` pattern (matching `version-bump.sh`'s
  own jq branch) so there is no in-place edit on the mounted FS; or
- Prefer `jq` consistently (the jq branch already uses temp + mv) and ensure the
  sed fallback is only a last resort; or
- Document `version-bump.sh` as a Linux/CI-only helper and have the release flow
  on Windows edit the four version points directly (the `shell-and-files` rule
  already favors Write/Edit over shelling out).

Low priority — releases are infrequent and the manual-edit workaround is quick.
